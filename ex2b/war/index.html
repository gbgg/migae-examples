This example keeps the ring rqst/resp adapter in the static (aot-compiled) servlet, and the handler function in an uncompiled namespace so it can be dynamically reloaded.

The drawback with this method is that you cannot dynamically rename your handler. That should not be much of a problem, though, since you will rarely need to do so, and if you do all you need do is stop the server, recompile the servlet, and restart the server.
